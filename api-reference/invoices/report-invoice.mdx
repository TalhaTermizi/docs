## Overview

Reports a **Simplified (B2C)** invoice to ZATCA. Unlike Standard invoices, Simplified invoices are issued first and then reported to ZATCA for compliance.

## Authentication

This endpoint requires a valid Bearer Token.

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

See the [Authentication](/authentication) guide.

---

## Endpoint

```http
POST /v1/e-invoicing/{onboard_id}/invoices
```

---

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `onboard_id` | string | Yes | Device onboarding identifier returned by the Onboard endpoint. |

# Request Body

## Top-Level Parameters

| Field | Type | Required | Description |
|------|------|----------|-------------|
| `invoice_number` | string | Yes | Unique invoice number generated by your ERP or application. |
| `invoice_type` | string | Yes | Invoice type. Use `simplified` for B2C invoices. |
| `issue_date` | string | Yes | Invoice issue date in `YYYY-MM-DD` format. |
| `actual_delivery_date` | string | No | Date the goods or services were delivered. |
| `issue_time` | string | Yes | Invoice issue time in `HH:mm:ss` format. |
| `note` | string | No | Additional note printed on the invoice. |
| `doc_currency_code` | string | Yes | Document currency (ISO 4217), e.g. `SAR`. |
| `tax_currency_code` | string | Yes | Tax currency (ISO 4217), usually `SAR`. |
| `customer_info` | object | Yes | Customer information. |
| `supplier_info` | object | Yes | Supplier information. |
| `payment_means` | string | Yes | Payment method such as `cash`, `bank_transfer`, or `credit_card`. |
| `payment_means_instruction_note` | string | No | Additional payment instructions. |
| `allowance_charge` | object | No | Invoice-level discount or surcharge. |
| `tax_total` | object | Yes | Invoice tax summary. |
| `legal_monitory_total` | object | Yes | Invoice monetary totals. |
| `invoice_lines` | array | Yes | List of invoice line items. |
