The Invoices API is the core of the eInvoices platform. You can use it to programmatically create and manage the full lifecycle of an invoice — from drafting through sending, viewing, and payment — and integrate that workflow directly into your own application or back-office systems.

## Base URL

```
https://api.einvoices.com/v1
```

## Authentication

Include your API key as a Bearer token in the `Authorization` header of every request.

```http
Authorization: Bearer YOUR_API_KEY
```

---

## The Invoice Object

Every endpoint that returns an invoice returns the same invoice object structure.

<ResponseField name="id" type="string">
  Unique identifier for the invoice, prefixed with `inv_`.
</ResponseField>

<ResponseField name="number" type="string">
  Human-readable invoice number (e.g. `INV-0042`). Auto-generated unless you supply `invoice_number` at creation.
</ResponseField>

<ResponseField name="status" type="string">
  Current lifecycle status. One of `draft`, `sent`, `viewed`, `paid`, `overdue`, or `void`.
</ResponseField>

<ResponseField name="client_id" type="string">
  ID of the client this invoice belongs to.
</ResponseField>

<ResponseField name="issue_date" type="string">
  The date the invoice was issued, in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="due_date" type="string">
  The date payment is due, in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="currency" type="string">
  Three-letter ISO 4217 currency code (e.g. `USD`).
</ResponseField>

<ResponseField name="subtotal" type="number">
  Sum of all line item amounts before tax and discounts.
</ResponseField>

<ResponseField name="tax_total" type="number">
  Total tax applied across all line items.
</ResponseField>

<ResponseField name="discount_total" type="number">
  Total discount applied to the invoice.
</ResponseField>

<ResponseField name="total" type="number">
  Final invoice total: `subtotal + tax_total - discount_total`.
</ResponseField>

<ResponseField name="amount_paid" type="number">
  Amount already paid against this invoice.
</ResponseField>

<ResponseField name="amount_due" type="number">
  Remaining balance: `total - amount_paid`.
</ResponseField>

<ResponseField name="line_items" type="array">
  Ordered list of line item objects. Each contains `id`, `description`, `quantity`, `unit_price`, `tax_rate`, and `amount`.
</ResponseField>

<ResponseField name="notes" type="string">
  Optional notes displayed on the invoice.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the invoice was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the most recent update.
</ResponseField>

## GET /v1/invoices

List all invoices in your account. Results are returned in reverse chronological order and support cursor-based pagination.

### Query Parameters

<ParamField query="status" type="string">
  Filter by invoice status. Accepted values: `draft`, `sent`, `viewed`, `paid`, `overdue`, `void`.
</ParamField>

<ParamField query="client_id" type="string">
  Return only invoices belonging to this client.
</ParamField>

<ParamField query="from_date" type="string">
  Return invoices with an `issue_date` on or after this date (`YYYY-MM-DD`).
</ParamField>

<ParamField query="to_date" type="string">
  Return invoices with an `issue_date` on or before this date (`YYYY-MM-DD`).
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of records to return per page. Maximum `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned by the previous response as `next_cursor`. Omit to start from the first page.
</ParamField>

### Example Request

```bash
curl https://api.einvoices.com/v1/invoices?status=sent&limit=5 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Example Response

```json
{
  "data": [
    {
      "id": "inv_a1b2c3d4",
      "number": "INV-0042",
      "status": "sent",
      "client_id": "cli_x9y8z7",
      "issue_date": "2024-06-01",
      "due_date": "2024-06-30",
      "currency": "USD",
      "subtotal": 1000.00,
      "tax_total": 100.00,
      "discount_total": 50.00,
      "total": 1050.00,
      "amount_paid": 0.00,
      "amount_due": 1050.00,
      "created_at": "2024-06-01T09:00:00Z",
      "updated_at": "2024-06-01T09:05:00Z"
    }
  ],
  "next_cursor": "cur_n3x7p4g3",
  "has_more": true
}
```

## POST /v1/invoices

Create a new invoice. Newly created invoices start in `draft` status. You can continue updating a draft invoice before sending it to your client.

### Request Body

<ParamField body="client_id" type="string" required>
  ID of the client to bill. The client must already exist in your account.
</ParamField>

<ParamField body="issue_date" type="string" required>
  Date the invoice is issued (`YYYY-MM-DD`).
</ParamField>

<ParamField body="due_date" type="string" required>
  Date payment is due (`YYYY-MM-DD`). Must be on or after `issue_date`.
</ParamField>

<ParamField body="line_items" type="array" required>
  One or more line items. Each item is an object with the following fields:

  <Expandable title="Line item fields">
    <ParamField body="description" type="string" required>
      A description of the product or service.
    </ParamField>
    <ParamField body="quantity" type="number" required>
      Quantity of units.
    </ParamField>
    <ParamField body="unit_price" type="number" required>
      Price per unit in the invoice currency.
    </ParamField>
    <ParamField body="tax_rate_id" type="string">
      ID of a saved tax rate to apply to this line item. Omit if the line item is not taxable.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="currency" type="string" default="USD">
  Three-letter ISO 4217 currency code for this invoice.
</ParamField>

<ParamField body="invoice_number" type="string">
  Custom invoice number (e.g. `INV-2024-001`). If omitted, eInvoices auto-generates a sequential number.
</ParamField>

<ParamField body="discount" type="object">
  An optional discount to apply to the invoice total.

  <Expandable title="Discount fields">
    <ParamField body="type" type="string" required>
      Either `fixed` (a flat currency amount) or `percent` (a percentage of the subtotal).
    </ParamField>
    <ParamField body="value" type="number" required>
      The discount amount. For `percent` type, provide a value between `0` and `100`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="notes" type="string">
  Optional notes or payment instructions displayed at the bottom of the invoice.
</ParamField>

### Example Request

```bash
curl -X POST https://api.einvoices.com/v1/invoices \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "client_id": "cli_x9y8z7",
    "issue_date": "2024-06-01",
    "due_date": "2024-06-30",
    "currency": "USD",
    "line_items": [
      {
        "description": "Web design services",
        "quantity": 10,
        "unit_price": 100.00,
        "tax_rate_id": "txr_v5t6u7"
      }
    ],
    "discount": {
      "type": "fixed",
      "value": 50.00
    },
    "notes": "Thank you for your business."
  }'
```

### Example Response

```json
{
  "id": "inv_a1b2c3d4",
  "number": "INV-0042",
  "status": "draft",
  "client_id": "cli_x9y8z7",
  "issue_date": "2024-06-01",
  "due_date": "2024-06-30",
  "currency": "USD",
  "subtotal": 1000.00,
  "tax_total": 100.00,
  "discount_total": 50.00,
  "total": 1050.00,
  "amount_paid": 0.00,

    {
```

## PUT /v1/invoices/{id}

Update an existing invoice. You can modify any field that was set during creation. All body parameters are optional — include only the fields you want to change.

<Warning>
  Only invoices in `draft` status can be updated. Attempting to update a `sent`, `paid`, or `void` invoice returns a `422 Unprocessable Entity` error. If you need to correct a sent invoice, void it and create a new one.
</Warning>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the invoice to update.
</ParamField>

### Request Body

All fields are optional. Provide only the fields you want to change. See [POST /v1/invoices](#post-v1-invoices) for full field definitions.

<ParamField body="client_id" type="string">
  Reassign the invoice to a different client.
</ParamField>

<ParamField body="issue_date" type="string">
  Updated issue date (`YYYY-MM-DD`).
</ParamField>

<ParamField body="due_date" type="string">
  Updated due date (`YYYY-MM-DD`).
</ParamField>

<ParamField body="line_items" type="array">
  Replaces the entire line items array. Any existing line items not included in the new array are removed.
</ParamField>

<ParamField body="currency" type="string">
  Updated ISO 4217 currency code.
</ParamField>

<ParamField body="discount" type="object">
  Updated discount. Pass `null` to remove an existing discount.
</ParamField>

<ParamField body="notes" type="string">
  Updated notes.
</ParamField>

### Example Request

```bash
curl -X PUT https://api.einvoices.com/v1/invoices/inv_a1b2c3d4 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "due_date": "2024-07-15",
    "notes": "Extended payment terms agreed upon on 2024-06-20."

```

## DELETE /v1/invoices/{id}

Permanently delete an invoice.

<Warning>
  Only invoices in `draft` status can be deleted. Sent or paid invoices cannot be deleted to preserve your audit trail — use the void action instead.
</Warning>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the invoice to delete.
</ParamField>

### Example Request

```bash
curl -X DELETE https://api.einvoices.com/v1/invoices/inv_a1b2c3d4 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Example Response

A successful deletion returns `204 No Content` with an empty body.

## POST /v1/invoices/{id}/send

Send an invoice to your client by email. Calling this endpoint transitions the invoice status from `draft` to `sent` and dispatches the email immediately. If no `to` address is supplied, the email goes to the primary address on the associated client record.

<Note>
  You can only send invoices that are in `draft` status. To resend an already-sent invoice, call this endpoint again — it will send a new email but will not change the status.
</Note>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the invoice to send.
</ParamField>

### Request Body

<ParamField body="to" type="string">
  Recipient email address. Defaults to the email on the associated client record.
</ParamField>

<ParamField body="subject" type="string">
  Email subject line. Defaults to `"Invoice [number] from [your business name]"`.
</ParamField>

<ParamField body="message" type="string">
  Custom body text for the email. Supports plain text. Defaults to your account's saved email template.
</ParamField>

<ParamField body="attach_pdf" type="boolean" default="true">
  When `true`, attaches a PDF copy of the invoice to the email.
</ParamField>

### Example Request

```bash
curl -X POST https://api.einvoices.com/v1/invoices/inv_a1b2c3d4/send \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "to": "billing@acme.com",
    "subject": "Invoice INV-0042 — payment due June 30",
    "message": "Hi Acme team, please find your invoice attached. Let us know if you have any questions.",
    "attach_pdf": true
  }'
```
