The Payments API lets you track every payment made against your invoices — whether a client paid online through a payment gateway or you received a bank transfer, check, or cash payment that you need to record manually. You can also issue refunds programmatically for any completed payment.

## Base URL

```
https://api.einvoices.com/v1
```

## Authentication

Include your API key as a Bearer token in the `Authorization` header of every request.

```http
Authorization: Bearer YOUR_API_KEY
```

---

## Online vs. Manual Payments

eInvoices supports two payment flows:

- **Online payments** — When a client clicks "Pay Now" on their invoice and completes payment through a connected gateway (e.g. Stripe), eInvoices automatically creates a payment record and marks the invoice as `paid`. You do not need to call any API endpoint to record these.
- **Manual payments** — When a client pays you outside the platform (bank transfer, check, cash), you record the payment yourself using `POST /v1/payments`. This updates the invoice balance and status accordingly.

<Note>
  Use `POST /v1/payments` exclusively for manual off-platform payments. Attempting to create a duplicate payment record for a gateway-processed payment will return a `409 Conflict` error.
</Note>

## The Payment Object

Every endpoint that returns a payment returns the same payment object structure.

<ResponseField name="id" type="string">
  Unique identifier for the payment, prefixed with `pay_`.
</ResponseField>

<ResponseField name="invoice_id" type="string">
  ID of the invoice this payment applies to.
</ResponseField>

<ResponseField name="amount" type="number">
  Payment amount in the invoice's currency.
</ResponseField>

<ResponseField name="currency" type="string">
  Three-letter ISO 4217 currency code matching the invoice.
</ResponseField>

<ResponseField name="status" type="string">
  Current payment status. One of `pending`, `completed`, `failed`, or `refunded`.
</ResponseField>

<ResponseField name="method" type="string">
  Payment method. One of `card`, `bank_transfer`, `check`, `cash`, or `other`. Online gateway payments report `card`.
</ResponseField>

<ResponseField name="gateway" type="string">
  Name of the payment gateway that processed the transaction (e.g. `stripe`). `null` for manually recorded payments.
</ResponseField>

<ResponseField name="date" type="string">
  Date the payment was received, in `YYYY-MM-DD` format.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the payment record was created.
</ResponseField>

## GET /v1/payments

Return a paginated list of payment records. Filter by invoice, status, or date range to narrow results.

### Query Parameters

<ParamField query="invoice_id" type="string">
  Return only payments applied to this invoice.
</ParamField>

<ParamField query="status" type="string">
  Filter by payment status. Accepted values: `pending`, `completed`, `failed`, `refunded`.
</ParamField>

<ParamField query="from_date" type="string">
  Return payments with a `date` on or after this value (`YYYY-MM-DD`).
</ParamField>

<ParamField query="to_date" type="string">
  Return payments with a `date` on or before this value (`YYYY-MM-DD`).
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of records to return per page. Maximum `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned as `next_cursor` in the previous response. Omit to start from the first page.
</ParamField>

### Example Request

```bash
curl "https://api.einvoices.com/v1/payments?invoice_id=inv_a1b2c3d4&status=completed" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Example Response

```json
{
  "data": [
    {
      "id": "pay_q1w2e3r4",
      "invoice_id": "inv_a1b2c3d4",
      "amount": 1050.00,
      "currency": "USD",
      "status": "completed",
      "method": "card",
      "gateway": "stripe",
      "date": "2024-06-15",
      "created_at": "2024-06-15T14:30:00Z"
    }
  ],
  "next_cursor": null,
  "has_more": false
}
```

## POST /v1/payments

Record a manual payment against an invoice. Use this endpoint when a client pays you outside the eInvoices payment gateway — for example, by bank transfer, check, or cash. eInvoices will apply the payment to the invoice balance and update the invoice status automatically.

<Note>
  If the payment amount equals the full `amount_due` on the invoice, the invoice status transitions to `paid`. Partial payments reduce `amount_due` but leave the status as `sent` or `viewed`.
</Note>

### Request Body

<ParamField body="invoice_id" type="string" required>
  The ID of the invoice this payment applies to. The invoice must be in `sent`, `viewed`, or `overdue` status.
</ParamField>

<ParamField body="amount" type="number" required>
  The payment amount in the invoice's currency. Must be greater than `0` and no more than the invoice's current `amount_due`.
</ParamField>

<ParamField body="date" type="string" required>
  The date the payment was received (`YYYY-MM-DD`).
</ParamField>

<ParamField body="method" type="string">
  How the payment was made. Accepted values: `bank_transfer`, `check`, `cash`, `other`. Defaults to `other` if omitted.
</ParamField>

<ParamField body="notes" type="string">
  Internal notes about this payment (e.g. a check number or bank reference). Not visible to the client.
</ParamField>

### Example Request

```bash
curl -X POST https://api.einvoices.com/v1/payments \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "invoice_id": "inv_a1b2c3d4",
    "amount": 1050.00,
    "date": "2024-06-15",
    "method": "bank_transfer",
    "notes": "Wire ref #TXN-20240615-001"
  }'
```

### Example Response

```json
{
  "id": "pay_q1w2e3r4",
  "invoice_id": "inv_a1b2c3d4",
  "amount": 1050.00,
  "currency": "USD",
  "status": "completed",
  "method": "bank_transfer",
  "gateway": null,
  "date": "2024-06-15",
  "notes": "Wire ref #TXN-20240615-001",
  "created_at": "2024-06-15T14:30:00Z"
}
```

## GET /v1/payments/{id}

Retrieve the full details of a single payment record by its ID.

### Path Parameters

<ParamField path="id" type="string" required>

</ParamField>

### Example Request

```bash
curl https://api.einvoices.com/v1/payments/pay_q1w2e3r4 \

```

```json
  "id": "pay_q1w2e3r4",
  "invoice_id": "inv_a1b2c3d4",
  "amount": 1050.00,
  "currency": "USD",
  "status": "completed",
  "method": "card",
  "gateway": "stripe",
  "date": "2024-06-15",
  "created_at": "2024-06-15T14:30:00Z"
}
```

## POST /v1/payments/{id}/refund

Issue a full or partial refund against a completed payment. For online payments processed via a gateway, eInvoices initiates the refund with the gateway automatically. For manually recorded payments, the refund is recorded in eInvoices only — you must arrange the actual funds transfer to your client separately.

<Warning>
  You can only refund payments with `completed` status. The refund `amount` cannot exceed the original payment amount. If the payment has already been partially refunded, the maximum refundable amount is `original_amount - previously_refunded_amount`.
</Warning>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the payment to refund.
</ParamField>

### Request Body

<ParamField body="amount" type="number" required>
  The amount to refund. Must be greater than `0` and no more than the original payment amount.
</ParamField>

<ParamField body="reason" type="string">
  The reason for the refund. Included in the refund record and, for gateway payments, passed to the payment processor.
</ParamField>

### Example Request

```bash
curl -X POST https://api.einvoices.com/v1/payments/pay_q1w2e3r4/refund \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 1050.00,
    "reason": "Client requested cancellation of services."
  }'
```

### Example Response

```json
{
  "id": "ref_r4t5y6u7",
  "payment_id": "pay_q1w2e3r4",
  "invoice_id": "inv_a1b2c3d4",
  "amount": 1050.00,
  "currency": "USD",
  "reason": "Client requested cancellation of services.",
  "status": "completed",
  "created_at": "2024-06-20T10:15:00Z"
}
```

<Note>
  After a full refund, the original payment's `status` transitions to `refunded` and the associated invoice's `amount_paid` is reduced accordingly. The invoice status reverts to `sent`.
</Note>
