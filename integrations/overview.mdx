eInvoices integrates with the tools your business already relies on — from accounting platforms like QuickBooks and Xero to payment gateways like Stripe, PayPal, and Square. You can also build custom workflows using the REST API and webhooks. All integrations are managed from a single place in your dashboard, so you spend less time on manual data entry and more time running your business.

## How to Connect an Integration

All integrations are enabled and managed from the **Settings → Integrations** page in your eInvoices dashboard. Each integration card shows its current status — *Connected*, *Disconnected*, or *Error* — so you always know what is active at a glance.

### OAuth-Based Connections (Accounting Software)

QuickBooks and Xero use the OAuth 2.0 authorization flow. When you click **Connect**, eInvoices redirects you to the provider's login page where you grant read/write access to your account. No passwords are ever stored in eInvoices — only a secure, revocable access token. You can disconnect at any time from either the eInvoices dashboard or directly from within QuickBooks or Xero.

### API Key-Based Connections (Payment Gateways)

Stripe and PayPal use API keys that you generate inside your payment provider's developer dashboard and paste into eInvoices. Square uses an OAuth flow similar to accounting software. Store your API keys securely and rotate them immediately if they are ever exposed.

<Note>
  You must be an **Owner** or **Admin** on your eInvoices account to add, edit, or remove integrations. Members with the **Billing** role can view integration status but cannot make changes.
</Note>

## What Data Syncs

Different integration categories exchange different types of data. The table below gives you a quick reference.

| Integration | Data Synced | Direction | Frequency |
|---|---|---|---|
| QuickBooks | Clients, invoices, payments, tax codes | eInvoices → QuickBooks | Real-time + manual resync |
| Xero | Contacts, invoices, payments, tax rates | eInvoices → Xero | Real-time + manual resync |
| Stripe | Payment status, transaction ID, refunds | Stripe → eInvoices | Real-time (webhook) |
| PayPal | Payment status, transaction ID, refunds | PayPal → eInvoices | Real-time (webhook) |
| Square | Payment status, transaction ID, refunds | Square → eInvoices | Real-time (webhook) |
| REST API | All resources | Bidirectional | On demand |
| Webhooks | Event payloads (invoices, payments, clients) | eInvoices → your endpoint | Real-time |

### Accounting Software

When you create or update an invoice in eInvoices, the changes are pushed to your connected accounting platform automatically. Client records are matched by email address to avoid duplicates. Tax codes you have configured in QuickBooks or Xero are imported into eInvoices during the initial sync so you can apply them to line items.

### Payment Gateways

Payment gateways sync in the opposite direction: when a client completes a payment through the hosted invoice page, the gateway sends a webhook to eInvoices. eInvoices then marks the invoice as **Paid**, records the transaction ID, and — if you have an accounting integration active — pushes the payment record to QuickBooks or Xero automatically.

### REST API and Webhooks

The REST API gives you full programmatic access to clients, invoices, payments, line items, and settings. Webhooks let you subscribe to specific events and receive a JSON payload at any HTTPS endpoint you control. Together, they allow you to embed eInvoices data into your own portals, CRMs, ERPs, or custom automations.

<Tip>
  If you use multiple integrations at once — for example, Stripe for payments and QuickBooks for accounting — eInvoices coordinates the data flow automatically. A payment collected through Stripe is pushed to QuickBooks as a received payment without any extra configuration on your part.
</Tip>

## Integration Health and Troubleshooting

Go to **Settings → Integrations** and click an integration card to see:

- **Last synced** timestamp
- A log of the most recent sync events and any errors
- A **Resync** button to manually trigger a full sync

If an integration shows an **Error** status, the most common causes are an expired OAuth token (reconnect the integration) or an invalid API key (re-enter the key from your payment provider's dashboard). Each error entry in the sync log includes a plain-language description of what went wrong and a suggested fix.
